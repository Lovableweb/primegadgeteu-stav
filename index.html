## PRVNÍ KROK KAŽDÉ SESSION
Než cokoliv uděláš, vždy nejdřív stáhni aktuální stav z GitHubu:
  git pull origin main
Pak si přečti soubor CLAUDE.md v kořeni projektu, ať víš, co se změnilo (i z mobilu). Teprve potom začni pracovat.

## OBOUSMĚRNÁ SYNCHRONIZACE CLAUDE.md — ZÁPIS PO KAŽDÉ ZMĚNĚ
Synchronizace tohoto souboru s GitHubem je OBOUSMĚRNÁ:
- **Stažení (na začátku):** viz „PRVNÍ KROK KAŽDÉ SESSION" výše — `git pull origin main`.
- **Nahrání (po každé změně):** kdykoliv něco dokončíš nebo upravíš stav projektu, automaticky to zapiš do `CLAUDE.md` (např. do sekce „Hotové funkce") a hned to nahraj na GitHub:
  git add CLAUDE.md
  git commit -m "docs: aktualizace CLAUDE.md"
  git push origin main
Tím zůstane GitHub verze (i ta upravená z mobilu) vždy synchronizovaná s lokální. `.env` se NIKDY necommituje.

## VEŘEJNÉ ZRCADLO PRO MOBIL (gist) — AKTUALIZOVAT PO KAŽDÉM PUSHI
Obsah tohoto souboru je zrcadlený do VEŘEJNÉHO gistu, aby šel načíst přes stálý raw odkaz (např. z mobilu). Po KAŽDÉ aktualizaci `CLAUDE.md` (po commitu a pushi na GitHub) je potřeba aktualizovat i tento gist:
  gh gist edit $(cat .claude/sync-gist.txt) CLAUDE.md
ID gistu je uložené v `.claude/sync-gist.txt`. Toto už dělá automaticky hook `.claude/hooks/push-claude-md.sh` — ručně jen pokud hook selže.
**POZOR:** gist je VEŘEJNÝ — do `CLAUDE.md` NIKDY nepatří hesla, API klíče ani tajemství.

## VEŘEJNÁ WEBOVÁ STRÁNKA (GitHub Pages) — REGENEROVAT PO KAŽDÉ ZMĚNĚ
Obsah `CLAUDE.md` je publikovaný i jako veřejná webová stránka přes GitHub Pages (dohledatelná vyhledávačem, čitelná z mobilu):
- **Adresa:** https://lovableweb.github.io/primegadgeteu-stav/
- **Hlavní repo `primegadgeteu` zůstává PRIVÁTNÍ.** Stránka je v SAMOSTATNÉM veřejném repu `Lovableweb/primegadgeteu-stav` (jen `index.html`), naklonovaném lokálně v `../primegadgeteu-stav`. Důvod: Pages na free plánu jdou jen na veřejných repech, a hlavní kód + Supabase migrace (admin email) nesmí být veřejné.

Po KAŽDÉ změně `CLAUDE.md` se stránka přegeneruje a pushne do veřejného repa:

## PRAVIDLA CHOVÁNÍ (NIKDY NEPORUŠUJ)

### 1. Žádné ptaní na zbytečnosti
Pokud uživatel říká "nastav mi Facebook reklamu na sluchátka" — UDĚLEJ TO.
Nevyptávej se na 10 věcí. Použij rozumné výchozí hodnoty. Ptej se maximálně na 1 věc.

### 2. Vždy kompletní výstup
Každý výstup musí být 100% připravený k použití:
- Kód → funkční, testovaný, s komentáři
- Reklamní texty → hotové, zkopírovatelné
- Emaily → kompletní HTML s předmětem
- Nastavení → přesný krok-po-kroku návod

### 3. Vždy česky
Komunikuj česky. Technické termíny vysvětli jednoduše.

### 4. Proaktivní návrhy
Po každém úkolu navrhni co by se dalo zlepšit nebo co dělat dál.

### 5. Startup kontrola (VŽDY na začátku session)
1. `git pull origin main` — stáhnout změny z GitHubu
2. Ověřit dev server na :8080, pokud neběží — spustit
3. Otevřít http://localhost:8080 v prohlížeči
4. Zkontrolovat `git status`

### 6. Po opravě chyby VŽDY prevence (ať se nikdy nevrátí)
Když najdu a opravím jakoukoliv chybu, NIKDY neskončím u pouhé opravy. Vždy ještě:
1. Opravím **příčinu**, ne jen příznak
2. Přidám **pojistku** (guard/kontrola), aby stejný stav nešel znovu nastat
3. **Zapíšu do paměti** (sem do CLAUDE.md + interní paměť): co se stalo a jak se to pozná příště
4. **Ověřím** opravu (test / build / smoke-test)
5. Pokud šlo o skript/hook/config → **zazálohuji do repa** (ne jen lokálně)
6. **Commit + push** na GitHub

### 7. Carousel / promo bannery — obsah VŽDY celý uvnitř plochy (mobil i desktop)
Slidy karuselu (`src/components/home/PromoCarousel.tsx`, `PS5PromoBanner.tsx`, `DronePromoBanner.tsx`) musí mít **veškerý obsah celý vidět uvnitř plochy na VŠECH šířkách** — nic se nesmí ořezávat ani přetékat za levý/pravý/horní/dolní okraj.
- **Výšku/poměr řeš JEDNOTNĚ na jednom místě** (viewport v `PromoCarousel`), ne v každém slidu zvlášť — ať se chyba nemůže vrátit u jednoho slidu.
- Poměr **5/2 jen na sm+** (široká obrazovka). Na **mobilu pevná vyšší výška**, aby se vešel obsah PS5 i celý obrázek dronu.
- **Drone banner je od 2026-06-21 ŽIVÝ** (HTML text, NE vypálený): obrázek `/drony-banner.webp` se ZÁMĚRNĚ ořezává tak, aby byla vidět jen pravá část s drony (vypálený text odříznut), levá strana = živý HTML text. Mobil: text nahoře / drony dole (img `w-[200%] object-right` → jen pravá polovina); sm+: text vlevo / drony vpravo (img `w-full object-right`). Okraje přelité do žluté (`#FBB816`). PS5 banner stále řeš dle potřeby.
- Texty/tlačítka **responsivně zmenšovat** na mobilu.
- **VŽDY ověřit na mobilní šířce (~375 px) i desktopu (~1440 px), oba slidy** — headless Edge screenshotem (viz [[pred-ui-zmenou-overit-regrese]]).
